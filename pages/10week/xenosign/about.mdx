# 회고 (KPT 방식 사용)

## KEEP

- 리액트 + FE 개발 씬에서 멀어진지가 좀 되었다고 생각하는데 최신 유행 기술(Vite, Vitest, MSW, Github 액션, 성능 최적화 등등)에 대해서 대략적으로 배울 수 있어서 좋았다
- 아 그런갑다 하면서 넘어갔던 부분들(tsconfig 의 설정, 그 외의 리액트 설정)에 대해서 전반적으로 배울 수 있어서 좋았다
- 리액트를 배웠지만 오히려 JS에 대한 이해가 늘어난 느낌
- 그래도 몇몇 새로운 리액트 훅을 배울 수 있었고, 기존에 사용하던 기술도 조금 더 이해할 수 있었다
- 리액트 훅 도입으로 컴포넌트 생명 주기를 너무 간략하게만 이해하고 넘어가는데 좀 더 심도있게 이해할 수 있었다
- 원치는 않았지만 SSR 관련 프레임워크를 간략하게만 알고 있었는데, 적어도 기본적인 내용은 알 수 있게 되었다
- 책과는 관련이 없지만 마크다운을 잘 안쓰는 옛날 사람이었는데 조금은 최신화가 되었네요 ㅋㅋㅋㅋㅋ

## PROBLEM

- 시장 상황 상 Next 가 들어가는 건 당연했지만, 책을 팔기 위해서인지 뜬금 없이 너무 빨리 나오기도 했고 분량도 너무 많아서 책의 전체 흐름이 아쉬웠다
- 책을 읽고 난 다음 리액트를 더 잘하게 되었으면 하는 마음으로 읽었는데, 막상 책이 끝나고 남은건 '리액트에 기반한 최신 웹 생태계의 이해' 라는 책을 읽은 느낌? 리액트를 다루는 법을 좀 더 심도 있게 다뤘으면 좋았을 것 같다
- 위에 추가해서 모든 저자가 그렇겠지만 분량에 있어서 저자가 잘 아는 부분은 과도하게 자세하고, 아닌 부분은 가볍게 넘어가는데 이런 부분의 밸런스가 잡혀있으면 더 좋았을 것 같다

## TRY

- FE 시장은 확실히 빠르게 변하고 있고, 빠르게 적응을 하는 것이 관건이 될 수 있겠다는 생각이 많이 들었네요. 개발자로서 뭐 계속 공부하는게 맞겠지만 FE 시장은 더 빡세질 거라 봅니다. 다들 공부합시다 ㅋㅋㅋㅋㅋ
- 책의 예제 코드를 보면서 확실히 나는 TS 를 잘 모른다는 확신이 들었네요. TS 도 공부 예정입니다
- 조금은 딴소리인데 최근 JAVA 를 배우다 보니 확실히 JS 가 가지는 자유도가 꼭 좋은 것은 아니구나라고 느꼈고, 이번에 JAVA 를 좀 잘배워서 JS 에도 개발 방법론을 적용해서 개발할 수 있는 개발자가 되어야겠다 다짐을 했습니다
- 많이 딴소리인데 아마도 높은 확률로 패배하겠지만 일단 알콜 배틀 한번 츄라이 해보겠습니다 ㅎㅎㅎㅎㅎ
