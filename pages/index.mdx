# 모던 리액트 Deep Dive 스터디

## 정해진 것

- 매주 일요일 오전 10시부터 12시까지
- 9명
- 스터디는 을지로 근처에서
- 오프라인

## 정해야 할 것

- 발표 어떻게 할 것인지
  - 순서를 정해서 매주 2명씩 발표를 할지 혹은 매주 짧게 짧게 다 발표를 할지 등등
- 스터디 기간 (8주,10주,12주)

## 스터디 진행 방식

- 매주 정해진 분량만큼 책 읽어오기
- 책을 읽으면 자유롭게 정리하기
  - 공부한 것을 정리 해올 필요는 없으나 매주 무조건 한 개 이상 이야기해 볼 것들을 준비해오기
  - 책을 읽고 몰랐던 것, 알게 된 것, 느낀 점 등등 자유롭게 이야기할것들
- 발표준비, 주제
  - 읽은 내용을 발표해도 되는데 책을 그대로 요약해서 발표하기 보단 개인의 스타일로 다시 정리하거나 혹은 공부하면서 느꼈던 점을 추가해서 발표하기
  - 책을 읽으면서 하나의 주제에 꽂혀서 그 주제에 대해서 공부해오고 자료를 찾아보고 그거 하나만 발표해도 됨
  - 책을 아무리 읽어도 이해가 안가는 부분이 있었는데 그거에 대해서 발표해도 됨
  - 자랑해도 됨
  - 코드로 문제 만들어와서 문제내도 됨
  - 회사 코드에 적용해서 가져와도 됨
  - 코드 짜와도 됨
  - 발표할 내용이 없으면 한 주 동안 뭐했는지 발표해도 됨
  - 이전 함수형 스터디에서는 책과 관련없는 내용에 대해서도 발표하고 그랬는데 이번에는 모든 스터디원분들이 프론트엔드 개발자이기도 하고 리액트에 대해서 더 이야기 나눠보고 깊게 공부했으면 좋겠다는 생각에 주제를 프론트엔드, 리액트, 자바스크립트 여러 라이브러리에 한정시킬까 합니다.!
- 스터디 규칙
  - 적극적인 소통 (잠수 x)
  - 싸우지 말기, 욕하지 말기
  - 지각 가능, 결석 가능 (말만 하기)
  - 벌금같은거 없음
  - 어렵지 않은 스터디, 그렇다고 의무적으로 책만 읽어오고 요약하고 그런 스터디는 지양하자
